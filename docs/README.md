# Documentation

| Document | Use it for |
| --- | --- |
| [Platform architecture](architecture.md) | Package boundaries, data flow, artifacts, commands, and timing invariants. |
| [Model specification](models.md) | Gas targets, features, weather inputs, model ladder, recursion, reconciliation, intervals, and promotion. |
| [Power fundamentals](power_fundamentals.md) | ERCOT hourly physical stack and implied gas-burn model. |

| [Decision log](decisions.md) | Historical design decisions and their tradeoffs. |

The architecture and model specification describe current behavior. The
decision log is historical and may reference structures that were later
replaced.

Update the architecture when package ownership, entry points, or artifacts
change. Update the model specification when targets, information sets, model
families, reconciliation, or promotion rules change.
